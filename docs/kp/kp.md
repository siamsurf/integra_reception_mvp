# Коммерческое предложение

## 1. Краткое описание проекта

INTEGRA Reception MVP — это прикладной AI-enabled сервис
для приёма, первичной проверки и маршрутизации клиентских заявок
в логистике.

Проект создан как рабочий MVP-инструмент,
который помогает сократить ручную обработку входящих обращений,
структурировать заявки и ускорить дальнейшую работу менеджера.

Решение уже поддерживает web-канал и VK-канал,
умеет принимать вложения, выполнять базовый precheck
и использовать OpenAI API как fallback
при сложных или неочевидных формулировках.

## 2. Для кого предназначен проект

Проект ориентирован на логистические компании,
импортёров и сервисные команды,
которые принимают входящие заявки из нескольких каналов
и тратят много времени на их ручную первичную обработку.

В первую очередь решение подходит для бизнеса,
где важно быстро понять суть запроса,
отфильтровать неполные или сомнительные обращения,
собрать базовые данные по грузу
и передать менеджеру уже более подготовленную заявку.

## 3. Что уже реализовано

На текущем этапе в проекте реализованы:

- web-интерфейс для подачи заявок;
- RU/EN toggle;
- административная панель;
- JSON export;
- attachment layer для работы с файлами;
- validation flags для спорных и подозрительных обращений;
- precheck по заявкам на доставку;
- fallback-обработка через OpenAI API;
- VK long poll канал;
- определение источника заявки (Web / VK);
- технический endpoint `/health`.

Проект не является просто макетом интерфейса:
основные сценарии уже работают и пригодны
для практического использования в формате MVP.

## 4. Используемые технологии

- Python
- FastAPI
- SQLite
- Jinja2
- OpenAI API
- VK Long Poll API
- HTML / templates
- Git / GitHub

Архитектурно проект выполнен в code-first подходе
и может быть далее развит в сторону более крупной прикладной системы.

## 6. Практическая ценность проекта

Практическая ценность проекта заключается в том,
что он решает реальную бизнес-задачу:
уменьшает долю ручной рутины
при приёме и первичной обработке клиентских заявок.

Сервис помогает:
- быстрее разбирать входящий поток обращений;
- приводить заявки к более структурированному виду;
- заранее выявлять неполные, спорные и подозрительные обращения;
- экономить время менеджера;
- создавать основу для дальнейшей автоматизации.

Дополнительная ценность проекта в том,
что нейросетевая модель и API здесь используются не формально,
а встроены в прикладной рабочий процесс.

## 7. Возможности дальнейшего развития

Следующим этапом проект может быть расширен
за счёт усиления антиспам-логики,
углубления бизнес-правил,
доработки обработки вложений
и подключения новых каналов приёма заявок.

Также возможны:
- развитие более глубокой AI-обработки входящих данных;
- добавление новых сценариев маршрутизации заявок;
- усиление ролевой модели;
- переход от MVP к более устойчивому production-решению.

В перспективе данный MVP рассматривается
как часть более крупной прикладной системы
для автоматизации приёма и первичной обработки
клиентских заявок в логистике.
